# Creative DNA Inferred Edge QA

The Creative DNA graph is an **idea map**, not a truth database.

Inferred edges are useful when they help retrieve a plausible adaptation path:

```text
reference post → reusable mechanism → owned account fit → concept seed → workflow route
```

They are not proof that a post should be made. Before production, every inferred
edge must still pass the target account's writing, design, image, source, and
copy approval gates.

## Current QA Focus

Review inferred edges around these high-centrality nodes first:

- `Workflow B New Post`
- `Post Concept Flow`
- `athlete-max`
- `athlete-stories`
- all `mechanism family → target account` edges
- all `mechanism family → possible post concept` edges

## Useful Inferred Edge Criteria

An inferred edge is useful when:

- The mechanism fits a real audience tension for the account.
- The proof surface can be changed without copying the source post.
- The account has a voice/design format that can carry the mechanism.
- The claim burden is manageable with available sources.
- The edge helps choose a workflow step or generate a better concept.

## Bad Inferred Edge Criteria

Remove or downgrade an inferred edge when:

- It only matches surface niche language.
- It depends on copying the original hook wording.
- It would force the account into an unapproved voice or design format.
- It creates product claims that are not in the claim bank.
- It points to Workflow B before the concept/copy is ready.

## Node-Specific QA

### Workflow B New Post

Workflow B should be downstream. It should receive concepts only after the angle
or concept is clear enough for slide-by-slide copy. If a concept is still broad,
route to `post-concept-flow` or `angle-variants` first.

### Post Concept Flow

Use this when the user gives a broad idea, avatar, pain point, desired
transformation, or content direction. It should not be the final copy engine; it
is the bridge from idea to approved concept architecture.

### athlete-max

Useful inferred edges must translate the mechanism into:

- mechanism-first performance explanation
- biological or behavioral cause → game-visible cost/fix
- clean, clinical, source-aware carousel format

Reject edges that create motivational, bro-science, hype, or unsupported claims.

### athlete-stories

Useful inferred edges must translate the mechanism into:

- fact-led athlete story
- origin, rivalry, pressure, private routine, or visualization arc
- late native product bridge only after the mental method is earned

Reject edges that become generic advice, clinical explanation, or forced product placement.
