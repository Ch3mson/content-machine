# Writing QA

Use this file before approving copy samples, `flow.md`, or final post copy.

## Quality Checklist

- [ ] The first line names a specific reader problem, desire, or tension.
- [ ] The copy follows `writing.md` locked rules.
- [ ] The copy reflects `writing/principles.md`.
- [ ] No banned pattern from `writing/banned-patterns.md` appears.
- [ ] The slide has one clear job.
- [ ] The copy fits the account `design.md` layout.
- [ ] Claims are sourced, marked for verification, or removed.
- [ ] Product mentions use only approved claims.

## Scoring

Rate each 1-10. Below the account threshold, revise before approval.

| Dimension | Question | Score |
| --- | --- | --- |
| Recognition | Would the target reader feel named? |  |
| Curiosity | Is there a reason to keep reading? |  |
| Specificity | Are the details concrete? |  |
| Rhythm | Does the copy move naturally? |  |
| Trust | Does it avoid hype and unsupported claims? |  |
| Fit | Does it fit the account design system? |  |
