# Hook And Pattern Usage Ledger

Use this file to track where a hook card, social-account post, or reusable
pattern has been adapted into your own accounts. This prevents accidental
overuse and makes cross-account reuse visible.

## Usage Table

| Date Used | Source ID | Source Type | Original Pattern | Target Account | Target Post | Adapted Hook | Reuse Mode | Result / Notes | Status |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  |  | hook-card / social-post / manual |  |  |  |  | direct-hook / pattern-only / visual-format / flow-structure |  | planned / drafted / posted / retired |

## Source ID Rules

- For hook cards, use the card slug, for example `2026-04-30-denis-averbukh-male-rules`.
- For social-account posts, use the stable ID from `../social-accounts/post-index.md`, for example `user-football-kingdubar--post-1`.
- For external manual ideas not yet saved, create a hook card first if the idea is likely to be reused.

## Reuse Modes

| Mode | Meaning |
| --- | --- |
| `direct-hook` | The adapted hook closely follows the source hook structure. Use sparingly. |
| `pattern-only` | The underlying mechanic was reused, but the hook wording changed heavily. Preferred. |
| `visual-format` | The layout or slide style was reused more than the copy idea. |
| `flow-structure` | The beat order was reused, but the topic and hook changed. |

## Review Rule

Before using a hook again, search this ledger for the source ID and the hook
archetype. If the same account has already used it recently, adapt a different
pattern or wait until the angle is meaningfully different.
