# Men's Fitness Account Playbook

This is the pilot account folder.

Use this account to turn competitor slideshow structures, design references, product context, and tone feedback into reusable source-of-truth files.

Product context: w(inner) is a mobile mental performance app for athletes. This account can borrow men's fitness aesthetics, but the content should focus on athletic performance under pressure rather than generic physique transformation.

## Workflow

1. Use `../../references/agents/account-onboarding.md` for the conversational onboarding process.
2. Keep the account vision and user feedback in `account-brief.md`.
3. Add competitor slideshow examples to `inputs/competitor-flow/`.
4. Add Pinterest or visual references to `inputs/design-images/`.
5. Add copy examples or tone notes to `inputs/copy-samples/`.
6. Show brief flow, design, and copy samples before finalizing rules.
7. Use `prompts.md` to extract raw flow and design principles into `extractions/`.
8. Review the raw extraction files.
9. Update `flow.md`, `writing.md`, and `image.md` with final account rules.
10. Generate one sample concept and run the QA checklist.

## Current Defaults

- Primary performance goal: saves and shares.
- Platform split: TikTok and Instagram share the same rules in v1.
- Product promotion: no hard in-slideshow CTA by default.
- Reference boundary: copy structure only, not exact wording, exact visual identity, or assets.
- Writing base: `references/skills/stop-slop/`.
- Research base: this account uses an athlete mental performance `sources.md`; future accounts should start from `../../references/templates/account-sources.md` and `../../references/research/README.md`.
- Product claim base: `../../product/app-brief.md` and `../../product/claim-bank.md`.
