# Writing QA

## Quality Checklist

- [ ] The first line names a specific reader problem, desire, or tension.
- [ ] The copy follows `writing.md` locked rules.
- [ ] The copy reflects `writing/principles.md`.
- [ ] The copy passes the three-persona check in `writing/refinement-personas.md`.
- [ ] No banned pattern from `writing/banned-patterns.md` appears.
- [ ] The draft uses the account-native list spine unless the user explicitly requested a story/arc.
- [ ] The slide has one clear job.
- [ ] The copy fits the account `design.md` layout.
- [ ] Claims are sourced, marked for verification, or removed.
- [ ] Product mentions use only approved claims.

## Scoring

Rate each 1-10. Below the account threshold, revise before approval.

Account threshold: 30/42 (≈7.1 average) for post-approval candidates. Below 30 means revise.

## Stop-Slop Quick Check

Before approving any copy, also run the stop-slop checks:

- Any adverbs? Kill them.
- Any passive voice? Find the actor, make them the subject.
- Inanimate thing doing a human verb ("the decision emerges")? Name the person.
- Sentence starts with a Wh- word? Restructure it.
- Any "here's what/this/that" throat-clearing? Cut to the point.
- Any "not X, it's Y" contrasts? State Y directly.
- Three consecutive sentences match length? Break one.
- Paragraph ends with punchy one-liner? Vary it.
- Em-dash anywhere? Remove it.
- Vague declarative ("The implications are significant")? Name the specific implication.
- Narrator-from-a-distance ("Nobody designed this")? Put the reader in the scene.
- Meta-joiners ("The rest of this essay...")? Delete. Let the essay move.
- Does the post drift into a mini-essay arc? Convert it into 3-5 list modules.
- Does the final slide only summarize? Replace it with the strongest final item.
